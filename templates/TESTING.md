# Testing Strategy

> This file is generated by `/pipeline-test-setup` during `/pipeline-init` or `/pipeline-build`.
> Do not copy this template directly — run the interactive setup to customize for your project.
> Edit the generated file as needed after setup.

## What This File Does

`.gsd/TESTING.md` tells the pipeline QA phase (`/pipeline-qa`) exactly how to test your project:

- **Platform** — web, mobile, desktop, api, cli, library
- **Test runner** — the exact command to run unit tests
- **Test types** — which types apply (unit, integration, e2e-browser, simulator, performance, security)
- **Environment** — env vars, setup/teardown commands
- **CI** — which CI system runs tests

## How It's Created

`/pipeline-test-setup` runs an interactive questionnaire:
1. Auto-detects your framework, test runner, and CI config
2. Asks which test types apply to your project
3. Asks follow-up questions per test type (URLs, commands, tools, env vars)
4. Writes `.gsd/TESTING.md` with exact commands for each test type
